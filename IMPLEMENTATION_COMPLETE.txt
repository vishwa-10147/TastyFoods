✨ WHAT HAPPENS AUTOMATICALLY
═══════════════════════════════════════════════════════════════════════════

Scenario 1: Customer Places Order
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Customer calls: POST /api/orders
   └─ Order created in database

2. Response sent to client immediately
   └─ No delay (async in background)

3. WhatsApp message triggers
   ├─ Format: Order Placed confirmation
   ├─ Content: Invoice with items, prices, total
   ├─ Send time: 1-2 seconds
   └─ Destination: Customer's WhatsApp

4. Customer receives message ✅
   └─ Shows order details & payment status


Scenario 2: Order Ready for Pickup
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Staff marks: POST /api/orders/123/status { "status": "ready" }
   └─ Status updated in database

2. Response sent to staff immediately
   └─ No delay (async in background)

3. WhatsApp message triggers
   ├─ Format: Order Ready notification
   ├─ Content: Pickup instruction
   ├─ Send time: 1-2 seconds
   └─ Destination: Customer's WhatsApp

4. Customer receives message ✅
   └─ Knows order is ready to pickup
